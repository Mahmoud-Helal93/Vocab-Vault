PART 3 — EXACT QUESTION GENERATION RULES

Create or refactor the question-generation system so each question type follows these exact rules.

Use a standardized question object/discriminated union if possible.

Each generated question should include:

- id
- type
- subtype
- wordId
- prompt
- options, if applicable
- correctAnswer or correctAnswers
- explanation
- belt
- mission
- set
- source example index if applicable
- part of speech if useful

#### Question types:

1. **Multiple Choice Questions**
    1. Word → Definition MCQ
    For each word:
        - create exactly 1 question
        - show the target word
        - show 4 definition options
        - user chooses the correct definition
            
            **Distractor rules:**
            
            - use same part of speech when possible
            - use same belt/mission/set when possible
            - avoid duplicate definitions
            - avoid obviously unrelated distractors when better ones exist
            - do not include the correct definition more than once
    2. Definition → Word MCQ
    For each word:
        - create exactly 1 question
        - show the definition
        - show 4 word options
        - user chooses the correct word
            
            **Distractor rules:**
            
            - use same part of speech when possible
            - use same belt/mission/set when possible
            - avoid duplicate words
            - avoid obviously unrelated distractors when better ones exist
    3. MCQ Synonym Recognition
    For each word:
        - if the word has 3 synonyms, create exactly 3 questions
        - each question tests one synonym
        - show the target word or a prompt like:
        “Which word is closest in meaning to ‘abound’?”
        - show 4 options
        - one option is the correct synonym
        - other options should be plausible distractors
        
        Use only synonyms stored in the learning-page data.
        Do not invent synonyms.
        
    4. MCQ Antonym Recognition
    For each word:
        - create exactly 1 question if the word has an antonym field/data
        - show 4 options
        - one option is the correct antonym
        
        Important:
        If the current data does not have antonyms, do not fake them.
        Instead:
        
        - create safe fallback logic
        - disable antonym question types
        - show “No antonym data available”
        - leave a clear TODO comment explaining where antonym data should be added
2. **Fill in the Blank**
For each word:
    - create 3 questions
    - each question comes from one of the word’s existing example sentences
    - do not use external examples
    - remove only the target word
    - user types the answer
    - no answer choices
    
    **Acceptance rules:**
    
    - ignore capitalization
    - ignore extra spaces
    - allow exact target word
    - only allow inflected forms if the sentence grammatically requires them
    - if the user enters a synonym, mark incorrect but explain that the target word was required
3. **Synonym Pairing / Select All Synonyms**
For each word:
    - create exactly 1 question
    - show the target word
    - show at least 6 option words
    - user selects all correct synonyms
    - all 3 stored synonyms should be correct answers if available
    - no partial credit
    - use only synonyms from the learning-page data
    - do not invent new synonyms
4. **True or False**
    1. True/False Definition
    For each word:
        - create exactly 1 question
        - statement tests whether a definition matches the word
        - include explanation after answer in Practice Mode
    2. True/False Synonym
    For each word:
        - create exactly 1 question
        - statement tests whether a synonym matches the word
        - use stored synonyms only
    3. True/False Antonym
    For each word:
        - create exactly 1 question if antonym data exists
        - if no antonym data exists, disable this type safely
    
    Remove:
    
    - T/F Tone
        - tone-based questions
        - any tone question chips from the question page
