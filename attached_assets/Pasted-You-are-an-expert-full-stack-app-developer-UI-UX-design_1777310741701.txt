QUESTION TYPES TO IMPLEMENT:

A. Foundational Questions

1. Multiple Choice Questions
Create two formats:
- Word-to-Definition Recall:
  The word is shown. Four definitions appear. User chooses the correct definition.
- Definition-to-Word Recall:
  The definition is shown. Four words appear. User chooses the correct word.

Add a toggle:
- Answer with term
- Answer with definition

Distractor rules:
- Wrong options must not be random.
- Use same part of speech when possible.
- Use similar difficulty level when possible.
- Prefer words from the same belt/mission/set when possible.
- Avoid obviously wrong distractors.
- Avoid duplicate or near-identical options.

2. Fill-in-the-Blank
For each word, create up to three fill-in-the-blank questions.
Each question must use one of the three example sentences already associated with the word in the Learn page.
Do not use external sentences.
Only remove the target word.
The user types the correct vocabulary word.
There are no answer choices.

Acceptance rules:
- Ignore capitalization.
- Ignore extra spaces.
- Allow reasonable inflected forms only when grammatically appropriate.
- If the user enters a synonym instead of the exact target word, mark it incorrect but show a helpful explanation.

3. Matching Synonyms / Antonyms
The target word is shown.
Six option words appear.
The user selects all correct synonyms or all correct antonyms.
There should be a toggle:
- Synonyms
- Antonyms

Use only synonyms and antonyms already stored in the Learn page data.
Do not invent new synonyms/antonyms unless the existing data is missing and the code clearly falls back safely.

Scoring:
- Practice Mode may show partial feedback.
- Test Mode gives no partial credit.

4. True / False
True/False questions can test:
- definition
- synonym
- antonym
- Arabic translation
- emotional tone if available

Use True/False lightly because guessing chance is high.
In Practice Mode, show explanation after answering.

B. GRE Simulation Questions

1. Text Completion
Create simulated GRE-style Text Completion questions.

Requirements:
- Passage length: 1 to 5 sentences
- Number of blanks: 1 to 3
- Single blank: 5 answer choices
- Multi-blank: 3 answer choices per blank
- Each blank has one unique correct answer
- Answer choices for different blanks function independently
- No partial credit in Test Mode

Question quality:
- The sentence or passage should test logic, not only memorization.
- Include context clues such as contrast, cause/effect, continuation, tone, or shift.
- Examples of clue words: however, although, despite, therefore, because, yet, nevertheless, moreover.

2. Sentence Equivalence
Create simulated GRE-style Sentence Equivalence questions.

Requirements:
- One sentence
- One blank
- Six answer choices
- User must select two correct answers
- The two correct answers must create equivalent meanings in the sentence
- They do not need to be perfect synonyms in every context
- No partial credit in Test Mode

Important:
If the current set does not contain a strong equivalent pair, the generator may pull one correct option from previously learned words or from the same belt, but it must avoid making the question feel forced.

TEST MODES:

1. Practice Mode
Practice Mode should be educational and supportive.

Requirements:
- No timer by default
- Immediate feedback after every question
- Show whether the answer is correct or wrong
- Show the correct answer
- Show a short explanation
- Explain why the correct answer works
- Explain why the selected answer is wrong when possible
- Show the related example sentence
- Allow retry for wrong answers where appropriate
- Include optional hint button
- Include “I don’t know” button
- Include “Add to difficult words” option
- Optionally ask confidence after answering:
  - I knew it
  - I guessed
  - I forgot it

2. Test Mode
Test Mode should feel more serious and exam-like.

Requirements:
- User chooses number of questions
- User chooses question types
- Questions should come from the belt level by default, not only mission or set level, so the test is more comprehensive
- No immediate feedback after each question
- Feedback appears only after the end of the test
- Timer for the whole test
- Track time per question
- Do not repeat the same question type for the same word in one session
- Use remaining question slots for other words
- Add flag-for-review button
- Add review screen before final submit
- Show answered, unanswered, and flagged questions
- Allow user to return to previous questions before submitting

ADAPTIVE REVIEW SYSTEM:
Add a mastery system for every word.

Each word should track:
- total attempts
- correct attempts
- wrong attempts
- accuracy
- last practiced date
- last incorrect date
- average response time
- confidence history if available
- mistake count
- difficult status
- mastery level
- next review date
- performance by question type

Suggested mastery levels:
- New
- Learning
- Weak
- Improving
- Strong
- Mastered

A word should not become Mastered after one correct answer.
A word becomes Mastered only when the user:
- answers it correctly multiple times
- answers it correctly across multiple question types
- answers at least one typed or context-based question correctly
- avoids recent mistakes
- answers within reasonable time

SPACED REPETITION:
After each answer or test session, update the word’s review schedule:
- wrong answer → review soon
- slow correct answer → review again soon
- fast correct answer → review later
- repeated correct answers → increase review interval
- low-confidence correct answer → still needs review
